## Overview

AI-powered trading agents consume market data feeds, news articles, social media sentiment, and economic indicators to make autonomous trading decisions. Attackers who can influence any of these data sources — through fake news injection, social media manipulation, or compromised data feeds — can cause the trading agent to execute losing trades, create predictable trading patterns that can be front-run, or trigger cascading liquidations.

## Attack Vector

1. Attacker identifies which data sources the trading agent monitors (news APIs, social feeds, data providers)
2. Injects false signals: fake breaking news, manipulated social sentiment, or spoofed data feed entries
3. Trading agent processes the poisoned data and adjusts its positions accordingly
4. Attacker takes the opposite position, profiting from the agent's predictable reaction
5. When the false signal is corrected, the agent incurs losses; attacker profits from the reversal

## Technical Details

```python
# Example: How a poisoned news feed affects an AI trading agent

# Attacker publishes fake news via a compromised or spoofed news API
fake_news = {
    "headline": "BREAKING: Fed announces emergency 50bp rate cut",
    "source": "reuters-alerts",  # Spoofed source identifier
    "timestamp": "2026-02-26T14:30:00Z",
    "confidence": 0.95
}

# AI trading agent's decision pipeline
class TradingAgent:
    def process_news(self, news_item):
        sentiment = self.llm.analyze(news_item["headline"])
        if sentiment["impact"] == "highly_bullish" and news_item["confidence"] > 0.8:
            # Agent goes long on bonds, short on dollar
            self.execute_trade("TLT", "BUY", self.max_position_size)
            self.execute_trade("UUP", "SELL", self.max_position_size * 0.5)
            # Attacker has already positioned for the reversal

    def execute_trade(self, symbol, side, size):
        # No human approval required — fully autonomous
        self.broker.place_order(symbol, side, size, order_type="MARKET")
```

**Financial-specific factors:**
- Trading agents operate at millisecond latency — no time for human verification
- Market impact is immediate and irreversible (especially with market orders)
- Multiple agents consuming the same feeds create herding behavior
- Regulatory liability falls on the agent operator, not the data source

## Impact

- **Direct financial losses** from erroneous trades executed at market speed
- **Cascading liquidations** if the agent triggers margin calls
- **Market manipulation liability** — operator may face regulatory action
- **Predictable exploitation** — attacker can repeatedly profit from the agent's reactions

## Mitigation

- Require multi-source confirmation for market-moving signals before trade execution
- Implement anomaly detection on incoming data feeds (sudden divergence from consensus)
- Use circuit breakers: maximum position change per time window, mandatory human approval above thresholds
- Cross-validate news against multiple independent sources before acting
- Maintain adversarial testing with fake signals in paper-trading environments
