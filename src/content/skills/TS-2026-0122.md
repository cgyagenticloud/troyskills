## Overview

Enterprise AI agents deployed in Slack and Teams (e.g., custom GPT bots, Copilot Studio agents) monitor channels and respond to queries. Any channel member can post messages that the agent will process as context. An attacker posts a message containing adversarial instructions disguised as normal conversation, which the agent ingests when responding to subsequent queries, causing it to act on the injected instructions with its full API permissions.

## Attack Vector

1. Attacker joins or is already a member of a Slack/Teams channel with an AI agent
2. Posts a message containing adversarial instructions hidden in formatting (code blocks, links, or Unicode tricks)
3. Another user asks the agent a question; the agent retrieves recent channel history as context
4. The attacker's message is included in the context window
5. Agent follows the injected instructions â€” leaking data, modifying settings, or spreading to other channels

**Enterprise-specific factors:**
- Agents often have OAuth tokens with broad workspace read access
- Channel membership may include contractors, partners, or former employees
- Message history is the primary context source for channel-based agents
- Agents can post to any channel they're invited to, enabling lateral spread

## Impact

- **Cross-channel data exfiltration** â€” agent leaks content from private channels
- **Worm-like propagation** â€” agent reposts injection payload to other channels
- **Social engineering at scale** â€” agent's messages are trusted by employees
- **Persistent** â€” message remains in channel history indefinitely

## Mitigation

- Implement message sanitization before agent context ingestion
- Use message-level permissions â€” agents should not inherit poster's intent as instructions
- Restrict agent API scopes to minimum necessary (no cross-channel read by default)
- Rate-limit and audit agent-initiated messages to external webhooks
- Deploy canary tokens in sensitive channels to detect unauthorized access
